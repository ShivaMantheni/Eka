Problem Statement: Laboratory Friction

error_outline Staging & QA Complexities
  • Manual Staging Delays: Traditional virtual machines and physical switches staging take hours of complex command-line entry.
  • Configuration Drift: Copying firmware versions manually and accessing hypervisors directly leads to error-prone deployments.
  • Tooling Fragmentation: Test developers rely on multiple programs (PuTTY, SCP file copy clients, terminal multiplexers).
  • Resource Collision: Multiple QA engineers attempting to connect or flash identical switches at the same time.

Solution: Unified Control Plane

check_circle Automated Orchestration Plane
  • Unified Control: Eka pools hypervisors, terminal SSH clients, and switch re-imaging loops into a central browser view.
  • Automated Provisioning: Asynchronous SCP copies and flashes firmware packages to targets automatically.
  • Direct Cable Topologies: Draw virtual Ethernet inter-interface cable connections directly on an execution canvas.
  • Heartbeat Locking: Background resource locking mechanisms block device connection overlap.

Eka Core Feature Highlights

star_outline Powering Modern QA Operations
  • Drag-and-Drop Topology: Draw cabling paths between switches on a GNS3-style interactive canvas.
  • ONIE Loop Detection: Identifies switches stuck in boot loops and flash firmware asynchronously.
  • Persistent Web SSH: Multiplex SSH console commands without third-party tools; persists commands in tmux sessions.
  • Asynchronous SCP Copying: Update VM and physical switches concurrently via multi-threaded download pools.
  • Logs Archival database: Search completed runs, logs output, and execution history.

Platform Benefits: Dev & QA Teams

trending_up Engineering Advantages
  • Drastic Time Savings: Automatic ONIE loops and hypervisor flash workflows save up to 45 minutes per setup cycle.
  • Simplified Access: Access switch CLI terminal sessions from any browser; zero VPN or client programs (PuTTY) required.
  • Zero Configuration Drift: Flashes clean builds and clears residual config files before dispatching runs.
  • Efficient Resource Pools: Visual device reservation indicators prevent hardware overlap and collision.

Scalability & Performance Metrics

speed Engineered for Velocity
  • Multi-Threaded Copy: Simultaneous SCP downloads flash up to 8 switches concurrently without speed drops.
  • FastAPI Async Engine: Lightweight heartbeats fetch status parameters with under 2% controller overhead.
  • Active Paramiko Pools: Re-uses backend SSH pools to connect and send CLI commands in under 100ms.
  • Distributed Docker Scaling: Server components can be scaled out in containers to manage hundreds of active nodes.

1. Dashboard Tab

dashboard Control Panel Overview
  • Monitor live connection states of all registered devices in real time.
  • Display visual counters for total devices, online nodes, and script queues.
  • Review run outcomes via recent execution telemetry charts.

2. Devices Tab

dns Inventory Management
  • Onboard physical switches and virtual machines with IP credentials.
  • Modify network parameter mappings (Ports, SSH credentials, paths).
  • Perform background ping heartbeats to determine device active states.

3. Execution Tab

play_circle_outline Interactive Topology
  • Draw cabling connections between virtual device interfaces on a canvas.
  • Browse hierarchical test script folders and select test groups.
  • Trigger automated SpyTest executions with a single click.

4. VS Manager Tab

settings_system_daydream Hypervisor Coordination
  • Query and display VM states from remote hypervisors.
  • Power on/off individual or groups of virtual switches instantly.
  • Fetch and copy VM build images from network storage via SCP.

5. Logs Tab

history Trace Logs Database
  • Access database audit logs for all completed and failed runs.
  • Expand execution rows to inspect live terminal stdio streams.
  • Filter records by date, script name, or test engineer.

6. Terminal Tab

terminal Integrated SSH Consoles
  • Open instant SSH terminal sessions to any registered device inside the UI.
  • Execute standard CLI commands without leaving the web browser.
  • Preserve console states using background tmux session persistence.

7. Hardware Load Tab

settings_input_hdmi ONIE Firmware Flasher
  • Update switch builds automatically via a single-button installer.
  • Auto-detect switches in ONIE boot loops and flash firmware packages.
  • View real-time console boot sequence logs during re-imaging.
